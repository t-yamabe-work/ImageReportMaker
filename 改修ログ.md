## 2026-04-21 worker2: 仮AppIcon整備

### 目的
- `/Applications/画像報告書メーカー.app` にアプリアイコンが反映される状態を作る
- Assets カタログ経由で Xcode ビルドに AppIcon を取り込む

### 実施内容
- `scripts/generate-icons.swift` を新規作成
  - NSGraphicsContext + CoreGraphics で青グラデ角丸背景＋白文字「画報」を描画
  - 16/32/128/256/512 の @1x / @2x（合計10枚）を一括PNG出力
  - Hiragino Sans W6 を使用（フォント未取得時は boldSystemFont フォールバック）
- `Apps/ImageReportMaker/Resources/Assets.xcassets/AppIcon.appiconset/`
  - `Contents.json`（macOS 用10エントリ）
  - 生成済みPNG 10枚を配置
- `Apps/ImageReportMaker/Resources/Assets.xcassets/Contents.json` を追加
- `project.yml`
  - targets.ImageReportMaker.info.properties に `CFBundleIconName: AppIcon` を追記
  - targets.ImageReportMaker.settings.base に `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` を追加
- `Apps/ImageReportMaker/Info.plist`
  - `CFBundleIconName = AppIcon` を追記

### 確認結果
- `xcodebuild ... Debug build` 成功
- `/Applications/画像報告書メーカー.app/Contents/Resources/` に `AppIcon.icns` と `Assets.car` 生成を確認
- `CFBundleIconName` = AppIcon、`CFBundleDisplayName` = 画像報告書メーカー が Info.plist に出力されていることを確認
- `scripts/build-and-install.sh` 実行で `/Applications/画像報告書メーカー.app` 再配置成功
