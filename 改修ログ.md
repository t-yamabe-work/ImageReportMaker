## 2026-04-21 v0.1.2 詳細折り返し＋150dpi（worker1）

### 目的
- フィードバック2点を Core に反映
  - W1-A: `→詳細` の右端見切れを解消（折り返し＋ハンギングインデント）
  - W1-B: 書き出し時の解像度を 150dpi に引き上げ（プレビューは 72dpi 維持）

### 公開 API 変更（worker3 は以下を参照して呼び出し側を修正）
- `RenderOptions` の **デフォルト dpi を 72 → 150** に変更
  - 既存呼び出し `RenderOptions(format: .png, dpi: 72.0)` は破壊なし（明示指定はそのまま通る）
  - 引数省略時は 150dpi となる
- `RenderOptions` に **factory** を追加：
  - `RenderOptions.export(format:jpegQuality:)` — 書き出し用プリセット（dpi=150, quality=0.92）
  - `RenderOptions.preview(format:)` — プレビュー用プリセット（dpi=72, quality=0.85、format デフォルト `.jpg`）
- 推奨移行：
  - プレビュー呼び出し → `RenderOptions.preview(format: .png)` または `.preview()`
  - 書き出し呼び出し → `RenderOptions.export(format: selectedFormat)`
  - これによりViewModel側の `previewDpi`/`exportDpi` 定数は撤去可能

### Core 実装
- `LayoutConstants.swift`
  - `caseDetailLineHeightMultiple: 1.25` を追加
  - `caseDetailOffsetPt: 38.13` を集約（既存ハードコード撤去）
- `Rendering/ReportRenderer.swift`
  - `wrapText(_:fontSize:weight:contentWidthPt:firstLineIndentPt:continuedIndentPt:)` を追加
    - `CTTypesetterSuggestLineBreak` で行ブレーク位置を計算
    - 1行目は `firstLineIndentPt = 0`、2行目以降は `continuedIndentPt = arrowWidthPt` のハンギングインデント
  - `measureLineWidth(text:fontSize:weight:)` を追加（行幅実測）
  - `PreparedLayout` を再設計
    - 固定の `caseLineHeightPt` / `caseDetailOffsetPt` を削除
    - `cases: [CaseBlock]` を保持。各案件に `titleBaselineYPt`、`detailFirstBaselineYPt`、`detailLineHeightPt`、`detailLines: [WrappedLine]`、`totalHeightPt` を持たせる
    - 複数案件は折り返し後の実高さで縦スタック
  - 描画: 折り返し済みの各行を `baseline + i × lineHeight` の位置に描画
- `Rendering/SVGExporter.swift`
  - 同じ `CaseBlock` を参照し、1行につき `<text>` 要素を1つ出力（x=leftPt+indent、y=baseline）

    - 並び替えロジックの真実は `@State private var draggedItem: URL?` に保持し、
      NSItemProvider はドラッグセッション成立用のダミー封筒のみ
  - `ReorderDropDelegate.validateDrop` で `draggedItem != nil` を要求して
    外部由来のテキストドロップを拒否する安全弁
  - `dropTarget.handleDrop(providers:)` の内部/外部分岐を、UTI フィルタから
    `draggedItem != nil` の存在チェックに切り替え（カスタム UTI を捨てたため）
